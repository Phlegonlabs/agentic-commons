# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities using GitHub Security Advisories (private reporting).

- Do not open public issues for exploitable vulnerabilities.
- Include reproduction steps, impact, and affected components.
- Include a minimal proof of concept when possible.

## Disclosure

1. Report privately through GitHub Security Advisories.
2. Maintainers triage and validate the issue.
3. A fix is prepared and released.
4. Public disclosure follows after a patch is available.

## Response Expectations

This project is maintained on a best-effort basis and does not provide SLA guarantees.
