# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| 0.1.x | Yes |
| < 0.1.0 | No |

## Reporting a Vulnerability

Please use GitHub Security Advisories (Private Vulnerability Reporting) for all security issues.

- Do not open public issues for exploitable vulnerabilities.
- Include reproduction steps, impact, and affected components.
- If possible, include a minimal proof of concept.

## Disclosure Process

1. Report privately via GitHub Security Advisories.
2. Maintainers triage and confirm severity.
3. A fix is prepared and validated.
4. Coordinated disclosure is published after a patch is available.

## Response Expectations

This project is maintained on a best-effort basis and does not provide SLA guarantees.

- Initial triage target: within 7 business days
- Status updates: as available during investigation and fix rollout
